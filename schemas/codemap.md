# schemas/

## Responsibility

The `schemas/` directory owns the **GSettings schema definition** for the Find My Mouse GNOME Shell extension. This single XML file (`org.gnome.shell.extensions.find-my-mouse.gschema.xml`) is the **authoritative registry of all user-configurable settings** — their names, types, defaults, ranges, and descriptions. The compiled binary (`gschemas.compiled`) is what GNOME Shell loads at extension init to materialize the `Gio.Settings` object.

This folder serves three roles:

1. **Schema Authority** — Every configurable property in the extension is declared here first. The TypeScript sources (`src/settings.ts`, `src/prefs.ts`, `src/extension.ts`, `src/spotlight.ts`, `src/spotlightEffect.ts`, `src/mouseTracking.ts`) are consumers and must stay in sync with this definition.
2. **GSI (GSettings Introspection) Contract** — The XML defines the D-Bus-accessible surface of the extension's configuration. Other tools (`gsettings`, `dconf-editor`, GNOME Control Center) can read/write these keys without touching the extension code.
3. **PowerToys Defaults Repository** — All default values are intentionally chosen to match Microsoft PowerToys Find My Mouse behaviour (documented per-key in descriptions).

Files in this directory:

| File | Role |
|------|------|
| `org.gnome.shell.extensions.find-my-mouse.gschema.xml` | Human-editable schema source (19 keys) |
| `gschemas.compiled` | Binary compiled schema (generated by `glib-compile-schemas`) |
| `codemap.md` | This file — architectural documentation |

## Design Patterns

### 1. GSettings Schema DSL (XML)

The schema uses GLib's XML schema language (`<schemalist>`, `<schema>`, `<key>`) to declare:

- **Key name** — kebab-case identifier (e.g. `spotlight-radius`)
- **Type** — GLib variant type signatures: `s` (string), `i` (int32), `d` (double), `b` (boolean), `as` (string array)
- **Default** — GVariant text-format default (e.g. `'#00000080'`, `100`, `9.0`, `true`, `[]`)
- **Range** — Optional `<range min="..." max="..."/>` for numeric types (e.g. `blur-radius` 0–50, `glass-opacity` 0–100)
- **Summary/Description** — Human-readable docs; descriptions reference PowerToys defaults for traceability

### 2. Singleton Schema Instance

The schema ID `org.gnome.shell.extensions.find-my-mouse` with path `/org/gnome/shell/extensions/find-my-mouse/` is a singleton — there is one global GSettings object per session. Both the extension process (`extension.js`) and the preferences dialog (`prefs.js`) obtain the same underlying storage via `this.getSettings()`.

### 3. Caching Layer with Lazy Re-Read (SettingsManager)

`src/settings.ts` implements a **SettingsManager** class that:
- Reads all 19 keys into typed, normalized cached properties at construction
- Normalizes hex colours into `[r,g,b,a]` float tuples (0–1 range) for Cairo/GLSL consumption
- Parss `rgba()` strings separately for `glow-color`
- Connects a single `changed::do-not-activate-gamemode` listener (others are wired in `extension.ts`)
- Exposes read-only getters for every cached value

Key design decision: **explicit caching rather than live lookups**. The original code used `settings.get_*()` calls every repaint frame; this was refactored to cache on construction + update on change signals. The `cacheSettings()` method is called from multiple signal handlers to stay coherent.

### 4. Signal-Driven Reactivity

Changes propagate via GSettings `changed::<key>` signals. Each key of interest gets a dedicated listener:

| Where wired | Keys listened to |
|-------------|-----------------|
| `SettingsManager` constructor | `do-not-activate-gamemode` |
| `FindMyMouseExtension.enable()` | `log-level`, `enable-glass-morphism`, `blur-radius`, `glass-opacity`, `glow-color`, `glass-tint`, `spotlight-ring-width`, `activation-method` |
| `FindMyMouseExtension.enable()` (generic) | `changed` (all keys — triggers full cache refresh) |

### 5. Two Rendering Pipelines, Same Settings Source

The schema keys feed **two rendering backends** that read from the same `SettingsManager` cache:

- **Cairo backend** (`spotlight.ts` `_onRepaint`): reads `cachedBgColorNormalized`, `cachedSpotlightColorNormalized`, `cachedRadius`, `cachedZoom` — uses Cairo `setSourceRGBA`, `arc`, `stroke` for a dimmed overlay + clear hole + circle ring
- **GLSL backend** (`spotlightEffect.ts` `vfunc_paint_target`): reads the same cached values plus glass morphism uniforms (`cachedBlurRadius`, `cachedGlassOpacity`, `cachedGlowColorNormalized`, `cachedGlassTintNormalized`, `cachedGlassMorphismEnabled`, `cachedRingWidth`) — passes them as `set_uniform_float` calls into the shell shader

### 6. Grouped Reset Buttons

The preferences UI (`src/prefs.ts`) implements **per-category reset buttons** (General, Appearance, Timing, Shake, Glass) that restore default values for each group. These hardcode the same defaults as the XML schema — a maintenance coupling that must be kept in sync.

### 7. Glass Morphism as Optional Feature Set

Seven keys are gated behind `enable-glass-morphism` (boolean): `blur-radius`, `glass-opacity`, `glow-color`, `glass-tint`, plus the pre-existing `background-color` and `spotlight-color` are also relevant. This is a **feature-flag pattern** where the GLSL shader checks `glassMorphismEnabled > 0.5` at runtime to toggle the blur/tint/glow pipeline vs a simple transparent centre.

## Data & Control Flow

### Settings Initialization Flow

```
GNOME Shell starts extension
         │
         ▼
Extension.enable()
         │
         ├── this.getSettings()           ← GSettings loaded from gschemas.compiled
         │         │
         │         ▼
         │   ├── settings.get_int('log-level')
         │   ├── setLogLevel(logLevel)
         │   ├── settings.connect('changed::log-level', ...)
         │   │
         │   ├── new GameModeClient()
         │   ├── new SettingsManager(settings)  ← reads ALL 19 keys into cache
         │   │         │
         │   │         └── cacheSettings()
         │   │               ├── get_string  → parseColor → normalize (×4 colours)
         │   │               ├── get_int     (×7 numeric keys)
         │   │               ├── get_double  (×4 double keys)
         │   │               ├── get_boolean (×2 boolean keys)
         │   │               └── get_string  → _parseRgbaString (×1 glow-color)
         │   │
         │   ├── new SpotlightManager(settingsManager)
         │   ├── new MouseTracker(settingsManager, callback)
         │   │
         │   └── Connect change listeners:
         │         ├── changed::enable-glass-morphism → cacheSettings + queueRepaint
         │         ├── changed::blur-radius           → cacheSettings + queueRepaint
         │         ├── changed::glass-opacity          → cacheSettings + queueRepaint
         │         ├── changed::glow-color             → cacheSettings + queueRepaint
         │         ├── changed::glass-tint             → cacheSettings + queueRepaint
         │         ├── changed::spotlight-ring-width   → cacheSettings + queueRepaint
         │         ├── changed::activation-method      → full re-setup of all modules
         │         └── changed (generic)               → cacheSettings
```

### Preference Change Flow

```
User changes setting in Preferences UI
         │
         ▼
Gtk.Widget event → settings.set_*() call
         │
         ▼
GSettings writes to dconf (binary on-disk store)
         │
         ▼
changed::<key> signal emitted on Gio.Settings object
         │
         ▼
Extension's signal handler fires:
   1. settingsManager.cacheSettings()   ← re-read value
   2. spotlightManager.queueRepaint()   ← trigger next frame
         │
         ▼
SpotlightManager._resetIdleTimeout()   ← restart the idle countdown
```

### Spotlight Activation Flow (runtime)

```
Activation trigger (shake/shortcut/click/always)
         │
         ▼
_toggleSpotlight() or _showSpotlight()
         │
         ├── Check GameMode (do-not-activate-gamemode + DBus state)
         ├── Check show-on-all-monitors
         │
         ▼
SpotlightManager.show(showOnAllMonitors)
         │
         ├── Create St.DrawingArea covering monitor(s)
         ├── Try SpotlightGLSLEffect → if fails, fall back to Cairo _onRepaint
         │
         ▼
Mouse moves → updateMousePosition(x, y)
         │
         ├── GLSL: setMousePosition → set_uniform_float → queue_repaint
         └── Cairo: queue_repaint → _onRepaint → get_context → cairo ops
         │
         ▼
Mouse stops → idle-timeout ms → hide()
```

## Integration Points

### 1. With `src/settings.ts` (SettingsManager)

| `spotlight-ring-width` | `cachedRingWidth` | int32 | spotlightEffect, prefs |

### 2. With `src/prefs.ts` (Preferences UI)

Every key is bound to an Adw widget (ComboRow, SpinRow, SwitchRow, ColorButton, ActionRow) in the 6-page preferences window:

- **General page**: activation method, Game Mode toggle, multi-monitor, reset
- **Appearance page**: background color, radius, zoom, ring color, ring width, reset
- **Glass page**: enable toggle, blur radius, opacity, glow color, glass tint, reset
- **Timing page**: idle timeout, animation duration, reset
- **Shake Detection page**: shake interval, shake sensitivity, reset
- **About page**: log level, about dialog

Two binding strategies are used:
1. **Manual read/write** (most keys) — read on page build, write via `connect("notify::value")`
2. **`settings.bind()`** (3 glass morphism keys) — two-way GSettings binding for `enable-glass-morphism`, `blur-radius`, `glass-opacity`

### 3. With `src/extension.ts` (Extension entry point)

The extension connects 10+ `changed::<key>` signals directly on the GSettings object. Generic `changed` signal triggers full `cacheSettings()` — this double-caching (once in SettingsManager constructor, once via signal) is a minor inefficiency but ensures correctness.

### 4. With `src/spotlight.ts` (Cairo rendering)

Reads `cachedBgColorNormalized`, `cachedSpotlightColorNormalized`, `cachedRadius`, `cachedZoom` in the `_onRepaint` callback. The zoom value controls the initial clear-hole size (zoom × radius) which then animates down to radius (animation is partly in Cairo, partly in the actor opacity).

### 5. With `src/spotlightEffect.ts` (GLSL rendering)

Reads all cached colour tuples, plus `cachedBlurRadius`, `cachedGlassOpacity`, `cachedGlowColorNormalized`, `cachedGlassTintNormalized`, `cachedGlassMorphismEnabled`, `cachedRingWidth`, and `cachedRadius` — feeds them as GLSL uniforms via `set_uniform_float()`. The GLSL shader uses these to compute:
- 5×5 Gaussian blur inside spotlight (when glass morphism enabled)
- Semi-transparent tinted overlay outside spotlight
- Ring border at spotlight edge
- Exponential glow falloff outside spotlight

### 6. With `src/mouseTracking.ts` (Shake detection)

Reads `cachedShakeInterval` (time window in ms) and `cachedShakeSensitivity` (percentage threshold). `detectShake()` uses the interval to filter old movement records and the sensitivity to compute the ratio of total path distance to bounding-box diagonal distance.

### 7. With `src/gamemodeClient.ts` (Game Mode integration)

The `do-not-activate-gamemode` boolean is checked before every spotlight toggle. The GameMode client monitors `com.feralinteractive.GameMode` over D-Bus and notifies the extension of state changes.

### 8. With `src/utils.ts` (Shared utilities)

Provides `parseColor()` used by SettingsManager to convert hex strings `#RRGGBBAA` to `[r,g,b,a]` byte tuples, and `setLogLevel()`/`debugLog()` used throughout — the log level is itself a schema key.

### 9. Build Integration

```bash
# Compile schema (required after any XML change)
glib-compile-schemas schemas/
# or
npm run build:schemas
```

The `npm run build:dist` pipeline includes `build:schemas` so the ZIP distribution always contains the latest compiled schema alongside the XML source.

### Colour Normalization Pipeline

All four colour keys (`background-color`, `spotlight-color`, `glow-color`, `glass-tint`) go through a two-step normalization in `SettingsManager`:

```
hex string (#RRGGBBAA)
    │
    ├─ parseColor()        → [r, g, b, a]  (0–255 bytes)
    │     in utils.ts
    │
    └─ / 255              → [r, g, b, a]  (0–1 floats)
         (or _parseRgbaString for glow-color)
```

The normalized float arrays are what `Cairo.setSourceRGBA()` and GLSL `set_uniform_float()` expect.

### Maintenance Checklist

When adding/modifying a schema key:

1. Edit `org.gnome.shell.extensions.find-my-mouse.gschema.xml` — add/change `<key>` element with correct type, default, range, summary, description
2. Run `glib-compile-schemas schemas/` (or `npm run build:schemas`)
3. Add cache property in `src/settings.ts`: private field, getter, read in `cacheSettings()`
4. Wire `changed::<key>` signal in `src/extension.ts` if the key needs runtime reactivity
5. Add UI widget in `src/prefs.ts` (ComboRow / SpinRow / SwitchRow / ColorButton) with proper read/write
6. Use the cached value in the consumer module (spotlight, mouseTracking, keybindings, etc.)
7. Update the appropriate reset button(s) in `src/prefs.ts` if the key belongs to a reset group
8. Update this table and the integration table in this file
